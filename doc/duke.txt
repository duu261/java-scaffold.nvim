1. INTRODUCTION                              *duke-introduction*

duke.nvim safely creates Maven, Gradle, and Spring Boot projects in
Neovim. It adds, updates, and removes root Maven dependencies,
opens generated Java source for an existing Java filetype or JDTLS setup, and
can pass a completed project to an optional handoff command.

Guided wizards collect destination, coordinates, package, Java, and
workflow-specific choices before a final review. Maven dependency workflows
provide single and multi-select pickers. Telescope provides searchable
pickers; native vim.ui remains the dependency-free fallback.

Neovim 0.11+ is the only core requirement. java, Maven, Gradle, curl, and tar
are needed only by their respective workflows. Telescope and external
handoff tools are optional.

Generators build inside private staging directories, validate expected build
files, and never overwrite an existing target. The plugin does not run,
format, or test projects, edit Gradle dependencies, or manage JDTLS.
Spring requests require HTTPS. Archives are inspected before extraction;
absolute paths, parent traversal, symlinks, and hardlinks are rejected.

2. CONFIGURATION                             *duke-configuration*

>lua
  require("duke").setup({
    group_id = "com.example",
    artifact_id = "demo",
    java_version = "auto",
    java_versions = {},
    java_homes = {},
    entry_selector = nil,
    maven = {
      command = "mvn",
      runner_java_version = "auto",
      wrapper = false,
      project_version = "1.0-SNAPSHOT",
      timeout = 180000,
      central_search_url = "https://search.maven.org/solrsearch/select",
      central_search_rows = 20,
      central_search_timeout = 15000,
      archetypes = {
        {
          name = "Maven quickstart",
          group_id = "org.apache.maven.archetypes",
          artifact_id = "maven-archetype-quickstart",
          version = "1.5",
        },
        {
          name = "Maven web application",
          group_id = "org.apache.maven.archetypes",
          artifact_id = "maven-archetype-webapp",
          version = "1.5",
        },
      },
    },
    gradle = {
      command = "gradle",
      runner_java_version = "auto",
      dsl = "kotlin",
      dsls = { "kotlin", "groovy" },
      languages = { "java", "kotlin", "groovy" },
      test_framework = "auto",
      timeout = 180000,
      default_project_type = "java-application",
      project_types = {
        { id = "java-application", name = "Java application" },
        { id = "java-library", name = "Java library" },
        { id = "java-gradle-plugin", name = "Gradle plugin" },
      },
    },
    spring = {
      metadata_url = "https://start.spring.io",
      dependencies_url = "https://start.spring.io/dependencies",
      starter_url = "https://start.spring.io/starter.tgz",
      project_type = "maven-project",
      language = "java",
      packaging = "jar",
      metadata_timeout = 30000,
      timeout = 60000,
    },
    handoff = {
      enabled = false,
      command = nil,
      required_executables = {},
    },
  })
<

setup() is optional. Defaults apply when it is not called.
All configured Spring Initializr URLs must use HTTPS. Curl permits only HTTPS
for original requests and redirects.

3. PUBLIC API                                      *duke-api*

setup([{opts}])                              *duke.setup()*
    Apply configuration. Calling setup() without options uses defaults.
    Every call clears cached Java runtime discovery.

new()                                         *duke.new()*
    Open the unified Maven, Gradle, and Spring Boot generator picker.
    new_maven(), new_gradle(), and new_spring() start those wizards directly.

create({kind}, {opts}, {callback})          *duke.create()*
    Create a project without UI. kind must be "maven", "gradle", or
    "spring". opts must contain:

      cwd             Existing destination parent directory.
      group_id        Maven group ID.
      artifact_id     Artifact ID and target directory name.
      java_version    Numeric target version string. "auto" is rejected.

    package_name is optional for every kind and otherwise derives from the
    coordinates.

    Maven options:

      archetype               Table containing group_id, artifact_id, and
                              version. Defaults to the first configured
                              maven.archetypes entry.
      version                 Project version. Defaults to
                              maven.project_version.
      wrapper                 Boolean. Defaults to maven.wrapper.
      command                 Defaults to maven.command.
      timeout                 Positive milliseconds. Defaults to
                              maven.timeout.
      runner_java_version     "auto" or numeric version string. Defaults to
                              maven.runner_java_version.

    Gradle options:

      project_type            Public Java project type. Defaults to
                              gradle.default_project_type.
      language                "java", "kotlin", or "groovy". Defaults to
                              "java" and combines with project_type.
      dsl                     Configured Gradle DSL. Defaults to gradle.dsl.
      test_framework          Defaults to gradle.test_framework.
      command                 Defaults to gradle.command.
      timeout                 Positive milliseconds. Defaults to
                              gradle.timeout.
      runner_java_version     "auto" or numeric version string. Defaults to
                              gradle.runner_java_version.

    Maven and Gradle runner selection creates a process-scoped JAVA_HOME and
    PATH. It never changes the target java_version or global environment.

    Spring options:

      boot_version     Required explicit Spring Boot version.
      dependencies     List of Initializr dependency IDs. Defaults to {}.
      project_type     "maven-project" or "gradle-project". Defaults to
                       spring.project_type and determines the build.
      language         "java", "kotlin", or "groovy". Defaults to
                       spring.language.
      packaging        "jar" or "war". Defaults to spring.packaging.
      name             Defaults to artifact_id.
      description      Defaults to an empty string.
      url              HTTPS URL. Defaults to spring.starter_url.
      timeout          Positive milliseconds. Defaults to spring.timeout.

    Creation never prompts, confirms, notifies success, changes cwd, edits a
    file, or invokes handoff. Success emits User DukeProjectCreated and calls
    callback once with:
>
      {
        ok = true,
        kind = "maven",
        project_dir = "/absolute/path/demo",
        entry_file = "/absolute/path/demo/src/main/java/.../App.java",
      }
<
    Failure calls callback once with ok = false, kind, and error, without an
    invented project path. Validation, adapter startup, process, and event-hook
    failures cannot produce a second callback. Event-hook and user-callback
    exceptions are logged.

add({opts}, {callback})                       *duke.add()*
    Add one root Maven dependency without search or UI. opts requires
    pom_path, group_id, artifact_id, and version. pom_path is explicit; no
    upward search occurs. scope defaults to compile and may be compile, test,
    provided, or runtime. Compile emits no scope element. Existing coordinates
    succeed without a change.

upgrade({opts}, {callback})               *duke.upgrade()*
    Upgrade one unique root Maven dependency. opts requires pom_path,
    group_id, artifact_id, and version, where version is the new explicit
    version. Managed, missing-version, property-backed, malformed, and
    ambiguous duplicate dependencies are rejected.

upgrade_parent({opts}, {callback})   *duke.upgrade_parent()*
    Upgrade the Spring Boot parent version. opts requires pom_path and
    version, where version is the new explicit parent version. A non-Boot
    parent, a missing parent, and a property-backed parent version are
    rejected. Skips the confirmation prompt shown by :DukeBootUpgrade; the
    API call itself is the confirmation.

remove({opts}, {callback})                 *duke.remove()*
    Remove one unique root Maven dependency. opts requires pom_path, group_id,
    and artifact_id. A missing coordinate succeeds without a change.

    Successful add(), upgrade(), upgrade_parent(), and remove() results have
    this shape:
>
      {
        ok = true,
        pom_path = "/absolute/path/pom.xml",
        changed = true,
        count = 1,
        saved = true,
      }
<
    Mutations read the latest POM immediately before editing. Unloaded files
    and clean loaded buffers are written and return saved = true. A loaded,
    already-modified buffer is updated in memory without forced write and
    returns saved = false. Write failures return ok = false.

outdated({opts}, {callback})             *duke.outdated()*
    Inspect literal explicit root Maven dependency versions sequentially.
    opts requires pom_path. Optional group_id and artifact_id filter one
    coordinate; provide both or neither. This operation never writes.
    Successful results add:
>
      {
        dependencies = {
          {
            group_id = "junit",
            artifact_id = "junit",
            current_version = "3.8.1",
            latest_version = "4.13.2",
          },
        },
        skipped = { managed = 0, property_backed = 0 },
        unchecked = 0,
        warning = nil,
        managing_parent = "spring-boot-starter-parent",  -- optional
      }
<
    Each dependency row may carry managed = true and a managing_parent field
    when the version was resolved from a parent or BOM rather than an explicit
    <version> element:
    dependencies contains only outdated rows. All-current succeeds with an
    empty list. A timeout or HTTP 429 after at least one completed lookup
    returns ok = true, partial rows, warning, and the exact unchecked count.
    Failure before the first completed lookup returns ok = false. Ambiguous
    duplicate coordinates are rejected before lookup.

    All six programmatic functions require a callback. Unknown option keys
    are ignored. Wrong types, missing required fields, unsupported enum
    values, and unknown project kinds return structured callback errors. Every
    result callback runs once on the Neovim main loop.

add_module({opts}, {callback})           *duke.add_module()*
    Add a module to an existing Maven reactor without UI. opts requires
    reactor_dir and artifact_id; reactor_dir must be an existing directory
    with a pom.xml, and there is no current-working-directory fallback.
    package_name is optional and otherwise derives from the reactor's
    groupId. Successful results have this shape:
>
      {
        ok = true,
        parent_pom = "/absolute/path/pom.xml",
        module_dir = "/absolute/path/child",
        rolled_back = false,
      }
<
    Failure returns ok = false and error. rolled_back is true only when the
    core restored the parent pom.xml after a failed save or promotion. It
    stays false both when nothing was written and when the restore attempt
    itself failed; the latter appends detail to error instead.

add_dependency()                     *duke.add_dependency()*
    Find the nearest pom.xml and start the same dependency workflow as
    :DukeAdd.

update_dependency()               *duke.update_dependency()*
    Find the nearest pom.xml and update one explicit root dependency version
    through the same workflow as :DukeUpgrade.

upgrade_boot_parent()           *duke.upgrade_boot_parent()*
    Find the nearest pom.xml and upgrade the Spring Boot parent version
    through the same workflow as :DukeBootUpgrade.

outdated_dependencies()       *duke.outdated_dependencies()*
    Find the nearest pom.xml and compare literal explicit root dependency
    versions through the same workflow as :DukeOutdated.

remove_dependency()               *duke.remove_dependency()*
    Find the nearest pom.xml and remove confirmed root dependency blocks
    through the same workflow as :DukeRemove.

new_module()                             *duke.new_module()*
    Start the :DukeModule wizard using the current working directory as the
    reactor directory.

clear_cache()                           *duke.clear_cache()*
    Delete all cached Initializr project metadata and dependency catalogs.
    Return true on success and false when deletion fails.

java_runtimes([{opts}])                *duke.java_runtimes()*
    Return the Java runtimes discovered for plugin and editor integration:
>
      {
        active = "23",
        homes = { ["23"] = "/path/to/jdk-23" },
      }
<
    active is the active Java major version, or nil when unavailable. homes
    maps Java major versions to JDK homes. Discovery stays cached until
    setup() runs or opts.refresh is true. Every call returns a deep copy.

select_runtime([{opts}])              *duke.select_runtime()*
    Select one discovered JDK. opts.min_version sets the minimum Java major
    version and defaults to 0. opts.prefer_active defaults to true. Active
    Java is selected when it is eligible and has a discovered home;
    otherwise, the lowest eligible version is selected. Return nil when no
    eligible JDK home exists. This function does not change environment or
    launch Java. A successful result has this shape:
>
      {
        version = "23",
        home = "/path/to/jdk-23",
        executable = "/path/to/jdk-23/bin/java",
      }
<

Successful creation emits User DukeProjectCreated. Event data
contains project_dir and entry_file. Handoff arguments may contain {project}
and {file}; without placeholders, the project path is appended.

==============================================================================
4. COMMANDS                                      *duke-commands*

Generators prompt for a destination parent directory, defaulting to the
current working directory. Before creation, a review shows the final
destination, coordinates, build system, Java target, runner JVM when
applicable, and workflow-specific settings. Canceling starts no generator
process. With Telescope, use <Tab> to toggle dependencies and <Enter> to
finish. Without Telescope, select dependencies through vim.ui one at a time,
then choose [Done].

                                                      *:DukeNew*
:DukeNew
    Choose Maven, Gradle, or Spring Boot, then start that workflow's wizard.

                                                    *:DukeMaven*
:DukeMaven
    Pick a Maven archetype, then prompt for destination, coordinates, package,
    and Java version. Review the project, then generate it. A single configured
    archetype skips the picker. When maven.wrapper is true, generate mvnw before
    promotion and require all Maven Wrapper files.

                                                    *:DukeSpring*
:DukeSpring
    Prompt for destination, coordinates, project name, description, package,
    Boot version, Maven or Gradle project type, language, packaging, Java
    version, and dependencies. Review the project, then download and extract
    the Spring Initializr project.

                                                    *:DukeGradle*
:DukeGradle
    Prompt for destination, coordinates, package, Gradle project type, source
    language, build-script DSL, and Java version. Review the project, then
    generate a wrapper-backed Java, Kotlin, or Groovy application, library, or
    Gradle plugin.

                                                  *:DukeModule*
:DukeModule
    Prompt for an artifact ID, then a package name pre-filled from the
    reactor's groupId, then confirm before adding a child module to the
    Maven reactor rooted at the current working directory. The parent
    pom.xml gains only the new <module> entry, created inside a fresh
    <modules> block when none exists. Canceling any prompt or the
    confirmation writes nothing. Only a root pom.xml with
    <packaging>pom</packaging> is eligible; a plain jar-packaging pom is
    rejected without creating a directory.

                                              *:DukeAdd*
:DukeAdd
    Find the nearest pom.xml. For Spring Boot, resolve selected Initializr
    dependency IDs. For plain Maven, search Maven Central and select results
    showing groupId:artifactId plus latest version. A single artifact opens a
    newest-first version picker followed by compile (default), test, provided,
    or runtime scope selection. Compile emits no scope element. Multi-select
    keeps shown latest versions and compile scope without another prompt.
    Insert non-duplicate dependency blocks. BOM, repository,
    annotation-processor, and Maven Central pom-packaging entries are excluded.
    Spring catalog and Maven Central rows already present in the root
    dependencies block carry [installed]. Marked rows remain selectable;
    structural duplicate detection remains the write guard.
    If Initializr no longer
    supplies an old Boot version's catalog, insertion requires a valid
    same-URL cache, compatible custom server, or Boot upgrade. Maven Central
    search needs live network access and has no offline fallback.

                                           *:DukeUpgrade*
:DukeUpgrade
    Find the nearest pom.xml and list root dependencies. Managed
    dependencies whose version comes from a parent or BOM are shown with
    their version resolved by `mvn dependency:list` and marked as managed;
    selecting one notifies about :DukeBootUpgrade instead of opening a
    version picker. :DukeUpgrade degrades to the prior hidden-with-count
    behavior when `mvn` is missing or the project fails to resolve. For
    explicit-version dependencies the behavior is unchanged: select one,
    then choose from Maven Central versions in newest-first order. The
    current version is marked and selecting it is a no-op. Property-backed
    versions are rejected with the property name. Only version text changes;
    scope and all other block content stay untouched. POM writes skip write
    autocommands, so a format-on-save chain cannot reformat the file around
    the edit and bury it in a whole-file diff. A manual :w still formats
    the buffer as configured.

                                       *:DukeBootUpgrade*
:DukeBootUpgrade
    Find the nearest pom.xml, resolve its <parent>, and refuse if it is not
    org.springframework.boot:spring-boot-starter-parent, if there is no
    <parent> at all, or if the version is property-backed. Otherwise list
    Maven Central's full release history for spring-boot-starter-parent in
    newest-first order; the current version is marked and selecting it is a
    no-op. An explicit confirmation, defaulted to cancel, shows current and
    target version before writing. The pom is re-read after the picker and
    before the write. Only the parent <version> text changes.

                                          *:DukeOutdated*
:DukeOutdated
    Find the nearest pom.xml and check root dependencies with literal
    explicit or managed versions sequentially against Maven Central. Managed
    versions are resolved by `mvn dependency:list` and marked with their
    managing parent when available. If `mvn` is missing or the project fails
    to resolve, managed dependencies degrade to the prior skipped-with-count
    behavior and explicit-version rows still work. Property-backed versions
    are skipped with counts. Timeout or HTTP 429 stops later lookups,
    preserves gathered rows, and reports the unchecked count. Rows show
    current -> latest. Selecting a managed row notifies about
    :DukeBootUpgrade and writes nothing. Selecting an explicit-version row
    enters the DukeUpgrade version picker and stale-file-safe write path.
    Canceling changes nothing.

                                           *:DukeRemove*
:DukeRemove
    Find the nearest pom.xml and multi-select root dependencies, including
    managed dependencies without versions. A mandatory confirmation lists
    every selected groupId:artifactId. Canceling or declining changes nothing.
    Confirming removes whole dependency blocks while preserving the root
    dependencies container, sibling blocks, comments, and surrounding blank
    lines.

                                               *:DukeInfo*
:DukeInfo [groupId:artifactId]
    Show latest and recent versions with release dates from Maven Central
    in a read-only scratch buffer. Without an argument, prompt for a
    coordinate. The buffer wipes on close.

                                               *:DukeClearCache*
:DukeClearCache
    Delete all cached Initializr project metadata and dependency catalogs.

                                                    *:DukeLog*
:DukeLog
    Open the in-memory operation log.

                                                    *:DukeHealth*
:DukeHealth
    Load a lazy plugin instance and run :checkhealth duke.

Run :checkhealth duke directly when the plugin is already loaded.

5. JAVA VERSIONS                                  *duke-java*

java_version selects the project compiler or toolchain target; "auto" uses
active Java. Maven and Gradle choices include active Java, JDK<version>
environment variables, java_versions, java_homes, and common JDK install
directories. A configured home is accepted only when bin/java reports the
version declared by its java_homes key.

Duplicate physical paths are probed once, each version probe is capped at one
second, and Maven or Gradle creation reuses one cached runtime snapshot.

Each build system has a separate runner_java_version that defaults to active
Java. This allows a modern Gradle JVM to target Java 8 or 11. Known runner
homes become scoped JAVA_HOME and PATH values; global shell state remains
unchanged. Wrappers may override JAVA_HOME, so health checks and project
creation report the detected runner Java.

Spring Boot versions and full-project Maven or Gradle types come from
Initializr metadata. Dependency choices use the selected Boot version.

6. OFFLINE METADATA                            *duke-offline*

Initializr project metadata and Boot-version dependency coordinates are
cached under stdpath("cache")/duke.nvim after successful requests,
separately for each configured Initializr URL. Fetch failures use those
files. Cached data supports dependency insertion offline; new project
archives still require network access.

Use :DukeClearCache when cached Initializr data becomes stale. The
next metadata request fetches fresh data.

Falling back to cache is never silent: one notification names why
(host unreachable, or the remote payload's schema was not recognized)
and how old the cached data is. The underlying error stays in
:DukeLog. The cache has no expiry and stale data is never refused;
the notification only informs the decision to keep going or clear it.

Initializr may stop serving dependency catalogs for old Spring Boot
versions. Insertion then requires a catalog for that version cached earlier
from the same configured Initializr URL. Without that cache, the Initializr
rejection is reported. Upgrade the project or configure an Initializr server
that still supplies the catalog.

The plugin does not add unsupported Boot versions to new-project choices or
bypass Initializr compatibility rules. Old-version lookup happens only when
:DukeAdd reads that version from an existing pom.xml.
Plain Maven poms instead use live Maven Central search. Dependency updates use
live Maven Central version lookup. The service may return HTTP 429; rerun later
because arbitrary searches and version lists are not cached.

7. SCOPE AND TROUBLESHOOTING              *duke-troubleshooting*

V1 owns Maven, Gradle, and Spring project creation, adding a module to an
existing Maven multi-module reactor, plus root-level Maven dependency add,
update, and removal workflows. It deliberately does not run applications,
format code, execute tests, edit Gradle dependencies, or manage JDTLS.

For failures:

1. Run :DukeHealth. This loads a lazy installation before invoking
   :checkhealth duke.
2. Run :DukeLog for process arguments and detailed failure context.
3. Check the executable required by selected workflow.
4. For old-Boot catalog rejection, use a same-URL cache, compatible custom
   Initializr server, or upgrade Boot.
5. Ensure custom Initializr URLs use HTTPS.
6. Retry Maven Central search after a reported timeout or HTTP 429 rate limit.
7. An existing-target error means the path belongs to the user. Choose another
   artifact ID or move the target; the plugin will not delete it.
